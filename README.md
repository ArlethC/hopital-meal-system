# 🏥 hospital meal system
Proyecto web desarrollado para integrarse a un sistema hospitalario existente, con el objetivo de digitalizar y centralizar el proceso de solicitud de alimentación para pacientes. Esto permitio reducir errores, mejorar la comunicación entre las áreas involucradas e implementar trazabilidad en el proceso.

Antes de su implementación, las solicitudes de comida, alergias, intolerancias y documentos nutricionales eran gestionados manualmente en papel entre las áreas de nutrición, enfermería y cocina.

## 🎯 Objetivo del proyecto

Automatizar el proceso de solictud de comida para los pacientes reemplazando formularios y comunicación en papel.

## 💻 Tecnologías utilizadas
- React js
- Tailwind css 
- Vite
- Node js
- Socket.io
- SQL Server

## ⚙️ Funcionalidades 
- Autenticación de usuarios
- Creación y modificación de solicitudes de alimentación para pacientes.
- Gestión de reclamos sobre la alimentación recibida, con soporte para adjuntar fotografías como evidencia.
- Generación e impresión de etiquetas con información del paciente y su dieta alimentaria.
- Gestión centralizada de alergias e intolerancias alimentarias de los pacientes.
- Configuración de horarios permitidos para la creación y modificación de solicitudes y reclamos.
- Pantallas en tiempo real para la visualización de información en cocina.
- Envío automático de correos electrónicos al crear solicitudes y registrar reclamos.
- Centralización de información nutricional y documentación compartida entre nutrición y cocina.
- Digitalización y automatización del flujo de comunicación entre nutrición y cocina.

## 👨‍💼 Responsabilidades

- Investigación del proceso de solicitud de alimentación para los pacientes
- Levantamiento de requerimientos mediante entrevistas y análisis de procesos y formatos existentes.
- Análisis del sistema existente y su integración  
- Desarrollo completo de la solución (frontend y backend)
